OpenSore investigates alerts by querying the same tools your engineers use. Connect your stack so OpenSore can pull logs, check recent deploys, map dependencies, and deliver findings where your team already works.

## Before you begin

Make sure you have:

- A OpenSore account — you can create one at [app.tracer.cloud/sign-up](http://app.tracer.cloud/sign-up)
- Access credentials for the tools you want to connect
- Permissions to create API keys

## Enterprise integrations

Enterprise integrations are configured through the OpenSore web app at [app.tracer.cloud](https://app.tracer.cloud). Once connected, they are available to all users in your organization.

**Observability.** OpenSore reads your logs, metrics, traces, and alerts to build investigation hypotheses. Connect [Grafana](/grafana) and [Datadog](/datadog).

**Infrastructure.** OpenSore maps your environment and dependencies across clouds and clusters. Connect [AWS](/aws).

**Communication.** OpenSore delivers investigation reports where your team already works. Connect [Slack, Discord, or Telegram](/messaging/index).

### Configure enterprise integrations

Each integration follows a simple, guided flow. Let's look at Datadog as an example.

1. Name - Helps distinguish between different instances
2. API Key - in Datadog, create a new API key and copy and paste into OpenSore.
3. Application Key - In Datadog, create a new Application Key and copy and paste into OpenSore.

## Local integrations

Local integrations are configured on the machine running OpenSore — via the CLI setup wizard or environment variables. They give OpenSore access to your databases, source control, error trackers, and workflow tools.

```bash
opensore integrations setup
```

Or set environment variables and OpenSore picks them up automatically.

| Category | Integrations |
| --- | --- |
| **Observability** | [Better Stack](/betterstack), [Honeycomb](/honeycomb), [Coralogix](/coralogix), [Sentry](/sentry) |
| **Databases** | [MongoDB](/mongodb), [MariaDB](/mariadb), [ClickHouse](/clickhouse), [Kafka](/kafka) |
| **Source control** | [GitHub](/github), [GitLab](/gitlab), [Bitbucket](/bitbucket) |
| **Kubernetes releases** | [Helm](/helm) |
| **Deployments** | [Vercel](/vercel) |
| **Orchestration** | [Prefect](/prefect) |
| **Incident management** | [OpsGenie](/opsgenie), [Jira](/jira) |
| **Reporting** | [Google Docs](/google-docs) |
| **AI coding assistants** | [OpenClaw](/openclaw) |

### Verify local integrations

After setup, confirm everything is working:

```bash
opensore integrations verify
```

To check a specific integration:

```bash
opensore integrations verify mongodb
```
