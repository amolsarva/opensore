---
title: "Mac Personal Agent Quickstart"
description: "Install OpenSore on macOS and configure a low-cost LLM plus WhatsApp or iMessage."
---

This path is for using OpenSore as a local personal agent instead of an incident-response demo.

## 1. Install

From the repo root while developing:

```bash
make install
uv run opensore personal doctor
```

For an installed release:

```bash
curl -fsSL https://install.opensore.com | bash
opensore personal doctor
```

## 2. Configure a low-cost LLM

Run:

```bash
uv run opensore onboard
```

Choose one of these starting points:

| Provider | Good first choice | Notes |
|---|---|---|
| Gemini | Google Gemini or Gemini CLI | Usually the cheapest hosted path to try first. |
| OpenAI | OpenAI or Codex CLI | Good general-purpose default. |
| Claude | Anthropic or Claude Code CLI | Strong reasoning, usually not the cheapest. |
| OpenRouter | OpenRouter | Useful when you want one key and multiple model options. |
| Ollama | Ollama local | No API key; speed depends on your Mac. |

Then verify:

```bash
uv run opensore doctor
```

## 3. Configure WhatsApp

OpenSore currently supports Twilio-backed WhatsApp delivery. Inbound WhatsApp message handling is on the personal-agent roadmap.

```bash
uv run opensore integrations setup whatsapp
uv run opensore integrations verify whatsapp
uv run opensore messaging pair --platform whatsapp
uv run opensore personal doctor
```

## 4. Configure iMessage

iMessage is not implemented as a native OpenSore channel yet. For now, use the OpenClaw bridge if you already run OpenClaw or a compatible local gateway:

```bash
uv run opensore integrations setup openclaw
uv run opensore integrations verify openclaw
uv run opensore messaging pair --platform imessage
uv run opensore personal doctor
```

Native macOS iMessage support needs a dedicated bridge because Apple does not provide a normal server-side iMessage bot API.

## 5. Start working

For the current repo, start with the interactive shell:

```bash
uv run opensore
```

For the personal-agent target, track the remaining work in [Personal Agent Roadmap](/personal-agent-roadmap).
